# User Journeys

## Overview

The user journey illustrates the complete interaction between a healthcare member and the Healthcare Claims Voice Agent. It covers the conversation from the initial greeting through authentication, intent handling, service execution, and conversation closure.

The journey is designed to provide a secure, efficient, and user-friendly experience while ensuring protected healthcare information is accessed only after successful member verification.

## Supported User Journeys

The Healthcare Claims Voice Agent supports the following member journeys:

- Claim Status
- Claim Submission
- Eligibility Verification
- Benefits Inquiry
- Provider Lookup
- Pre-Authorization Status
- Service Request
- Human Agent Escalation

---

## Journey Outcome

A successful user journey ensures that:

- The member is securely authenticated.
- The correct healthcare service is identified.
- The appropriate backend service is invoked.
- Accurate information is returned to the member.
- Additional assistance is offered before ending the conversation.
- Unresolved requests are escalated when required.
