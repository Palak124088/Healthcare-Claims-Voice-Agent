# User Personas

## Overview

The Healthcare Claims Voice Assistant serves three primary user groups within the healthcare ecosystem. Each persona has unique goals and interaction requirements.

| Persona | Description | Primary Goals |
|----------|-------------|---------------|
| **Healthcare Member** | Member enrolled in a healthcare plan | Check claim status, eligibility, benefits, provider lookup, pre-authorization |
| **Healthcare Provider** | Hospital or clinic interacting on behalf of a member | Verify coverage, pre-authorization, provider information |
| **Support Representative** | Human agent handling escalated conversations | Resolve disputes, authentication failures, and complex requests |

## Authentication

Members must provide:

- Member ID
- Date of Birth
- ZIP Code
- Last 4 digits of SSN

Authentication is required before accessing Protected Health Information (PHI).

## Key Takeaway

These personas guide conversation design, authentication workflows, and intent routing while ensuring secure and personalized healthcare interactions.
